## Problem

Agent dispatch in Clavain is hand-coded: flux-drive uses LLM triage scoring to pick agents, sprint uses a hardcoded phaseâ†’command map. Neither reads the agency spec (C1) or fleet registry (C2), so the declared agent roster and capability model are ignored at runtime. Adding or removing agents requires editing skill prompts, not config files.

## Solution

A `clavain-cli compose` Go subcommand that joins agency spec stage requirements against the fleet registry, applies Interspect calibration for model selection, checks budget, and emits a JSON dispatch plan on stdout. Consumers (flux-drive, sprint) iterate the plan instead of hand-coding agent lists.

## Features

### F1: Go Fleet Registry Reader
**What:** Parse `fleet-registry.yaml` in Go, exposing typed agent queries (by role, by capability, cost estimate).
**Acceptance criteria:**
- [ ] Go struct mirrors fleet-registry.yaml schema (agent ID, capabilities, roles, models, cold_start_tokens, runtime)
- [ ] `fleetByRole(role) â†’ []Agent` returns agents whose `roles[]` contains the given role

- [ ] Reads from `CLAVAIN_CONFIG_DIR/fleet-registry.yaml` with fallback to `os/clavain/config/fleet-registry.yaml`
- [ ] Table-driven Go tests with fixture registry files

### F2: Go Agency Spec Reader
**What:** Parse `agency-spec.yaml` (with project-level override merge) in Go, exposing per-stage agent rosters and budget shares.

### F3: Compose Algorithm
**What:** The core join: match spec roles to fleet agents, apply Interspect calibration for model tiers, check budget, emit plan.
**Acceptance criteria:**
- [ ] `clavain-cli compose --sprint=<id> --stage=<stage>` produces JSON dispatch plan on stdout
- [ ] For each required role in spec: finds matching fleet agents via `fleetByRole()`, selects one (prefer agent with lowest cold_start_tokens if multiple match)
- [ ] For each optional role: fills from fleet if budget allows
- [ ] Validates that selected agents collectively cover all `spec.requires.capabilities` via `fleetCheckCoverage()`
- [ ] Reads Interspect calibration (`routing-calibration.json`): if confidence >= 0.7 and evidence_sessions >= 3, uses `recommended_model` instead of fleet `preferred` model
- [ ] Reads routing overrides (`routing-overrides.json`): agents with `action: "exclude"` are removed from candidates, logged in warnings
- [ ] Reads sprint stage budget via `ic run tokens <run_id>` + stage share calculation (existing Go budget math)
- [ ] If estimated total exceeds budget: adds `"budget_exceeded"` to warnings, still emits full plan
- [ ] If a spec role has no fleet candidates: adds `"unmatched_role:<role>"` to warnings
- [ ] Safety floors: `fd-safety` and `fd-correctness` are never assigned a model below `sonnet`, regardless of Interspect calibration
- [ ] Output JSON schema matches brainstorm spec: `{stage, sprint, budget, estimated_total, warnings, agents[]}`
- [ ] Each agent entry: `{agent_id, subagent_type, model, estimated_tokens, role, required, model_source}`
- [ ] `model_source` is one of: `interspect_calibration`, `fleet_preferred`, `safety_floor`, `routing_fallback`
- [ ] Table-driven Go tests covering: happy path, budget exceeded, excluded agent, unmatched role, safety floor enforcement, empty calibration

### F4: Consumer Wiring
**What:** Wire flux-drive and sprint pipeline to consume Composer output instead of hand-coded dispatch.
**Acceptance criteria:**
- [ ] `lib-compose.sh` thin shell wrapper: `compose_dispatch() { clavain-cli compose --sprint="$1" --stage="$2"; }`
- [ ] flux-drive launch phase (Step 2.0.5): calls `compose_dispatch` instead of `routing_resolve_agents`, iterates plan for Agent() calls
- [ ] sprint pipeline: calls `compose_dispatch` before each phase dispatch, passes plan to the executing skill
- [ ] Existing `routing_resolve_agents` still works for non-Composer callers (backward compatible)
- [ ] If `clavain-cli compose` fails (exit 1), callers fall back to existing hand-coded dispatch with a warning
- [ ] Integration test: end-to-end sprint with Composer-driven dispatch produces valid agent launches

## Non-goals

- **Budget optimization:** No automatic agent dropping or model downgrading. Warn-and-emit only.
- **Codex dispatch integration:** The Composer's `model` field targets subagent tiers (haiku/sonnet/opus). Codex dispatch uses a separate `--tier fast|deep` namespace. Bridging these is a follow-up.
- **Dynamic fleet discovery:** Fleet registry remains manually maintained YAML. Auto-discovery from installed plugins is a separate feature.
- **Full lib-fleet.sh/lib-spec.sh deprecation:** Those shell libraries continue working. We add Go readers alongside them, not replace them yet.

## Open Questions
