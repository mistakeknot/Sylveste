# Sylveste Brand Architecture

This is internal launch doctrine for keeping the public Sylveste surface legible.

## Public launch rule

The pre-Mythos public surface names **Sylveste**, **Clavain**, **Skaffen**, **Intercore**, **Interverse**, **Autarch**, **Interspect**, and the relevant `inter-*` capabilities when they are needed to explain the mechanism.

Do **not** introduce Garden Salon or Meadowsyn on the public Sylveste launch surface until each has its own shipped surface. They are real parts of the architecture, but naming them before they are legible dilutes the Sylveste story and creates unnecessary brand load.

## Brand roles

| Brand | Register | Role | Public launch posture |
|---|---|---|---|
| Sylveste | SF / infrastructure | Agent orchestration, evidence, routing, review, and launch-readiness platform | Primary public surface |
| Clavain | SF / operating rig | Reference Claude Code workflow and disciplined multi-agent engineering rig | Public and linked from Sylveste |
| Skaffen | SF / runtime | Sovereign Go agent runtime and OODARC execution loop | Public once it has a README/front door |
| Garden Salon | Organic / collaboration | Human-agent shared project space and experience layer | Hidden from Sylveste public launch copy until shipped |
| Meadowsyn | Bridge / visualization | Real-time visualization and systems dashboard layer | Hidden from Sylveste public launch copy until shipped |

## Copy boundary

Use this public hierarchy:

1. **Sylveste** — what the whole system does: orchestrates agents by human/machine comparative advantage.
2. **Clavain** — how a Claude Code user experiences the disciplined workflow today.
3. **Skaffen / Intercore / Interverse / Interspect** — mechanism details when the reader needs architecture.
4. **Garden Salon / Meadowsyn** — internal roadmap context only until those products launch.

## Enforcement checklist

Before publishing Sylveste launch copy:

- README hero uses the locked claim: “Sylveste orchestrates agents by human/machine comparative advantage.”
- Mission copy states the infrastructure thesis, not the product-family architecture.
- Public pages do not mention Garden Salon or Meadowsyn unless the page is explicitly about those products.
- Plugin lists are linked as standards/graphs, not dumped into the hero page.
- Architecture tables stay below the product claim and workflow explanation.
