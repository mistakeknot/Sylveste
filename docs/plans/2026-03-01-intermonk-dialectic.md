**Key adaptations from source:**

1. **Agent tool spawning (Phase 3):**
   ```
   Source: echo "[PROMPT]" | claude -p --allowedTools web_search,web_fetch,read_file > monk_a_output.md
   Adapted: Use Agent tool with subagent_type="general-purpose", run_in_background=true for parallel monks.
            Orchestrator writes returned text to dialectics/[topic]/monk_a_output.md
   ```

2. **Parallel execution:**
   ```
   Source: Shell background jobs
   Adapted: Multiple Agent tool calls in single message. Both monks can run concurrently.
   ```

3. **Validation agents (Phase 6):**
   ```
   Source: Resume same claude -p session
   Adapted: Fresh validation agent with summary of monk's original argument.
            The Agent tool's resume parameter exists but persona persistence
            is unreliable. Default to fresh agents with context summaries.
   ```

4. **Model selection:**
   ```
   Source: --model flag
   Adapted: model parameter on Agent tool. Default: inherit parent.
            Document heterogeneous model option in AGENTS.md.
   ```

5. **File output convention:**
   Add explicit orchestrator file-writing steps after each agent returns. Structure:
   ```
   dialectics/[topic-name]/
   ├── context_briefing.md
   ├── monk_a_output.md
   ├── monk_b_output.md
   ├── determinate_negation.md
   ├── sublation.md
   └── dialectic_queue.md
   ```

6. **Phase re-reading for recursion (Phase 7):**
   ```
   Source: grep/sed XML tags to re-read phases
   Adapted: Reference companion files + re-read relevant SKILL.md sections.
            XML tags preserved for selective extraction.
   ```

**Context management (from architecture review):**
- Monks must write their output to disk AND return it. The orchestrator writes the returned text to `dialectics/[topic]/monk_X_output.md`.
- For Phase 4 (determinate negation), the orchestrator reads monk artifacts from disk — targeted selective reads, not full ingestion.
- `auditor-prompt.md` is a **required** operational artifact, not optional reference. Phase 6 cannot complete without it. Mark this clearly in SKILL.md.

**Structural notes:**
- Keep XML phase tags (`<phase1>`, `<phase2>`, etc.) for selective re-reading during recursion
- Each phase should include clear USER CHECKPOINT markers where the orchestrator must stop and ask
- Anti-hedging language must be preserved verbatim — it's functionally critical

**Done when:** `/intermonk:dialectic` is invocable and the skill reads coherently as an orchestration protocol. Structural tests pass.
