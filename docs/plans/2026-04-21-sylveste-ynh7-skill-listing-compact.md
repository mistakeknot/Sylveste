# sylveste-ynh7 Implementation Plan — skill_listing + deferred-tools compact

> **For Claude:** REQUIRED SUB-SKILL: Use clavain:executing-plans to implement this plan task-by-task.

**Bead:** sylveste-ynh7
**Goal:** Shrink the on-wire skill_listing attachment (≤10KB from 39KB) and deferred-tools delta (≤8KB from 17KB), delivering a measurable in-scope preamble reduction per session, with no user-facing behavior regression.

**Success targets (tiered):**
- **Acceptable (in-scope floor):** ≥500 tokens saved from Sylveste-owned skill trims + external plugin disables. Bead closes as complete.
- **Partial-success (1200–1999 tokens):** close bead as partial + file follow-up referencing sibling bead sylveste-49kl for the Agent-schema + SessionStart levers that carry the remaining ~1500 tokens toward the cost-breakdown doc's 2000-token projection.
- **Stretch (≥2000 tokens, combined with sylveste-49kl):** the original 2000-token figure assumed Targets #1 + #2 + #3 combined; this bead owns Target #1 and contributes one-third of the stack. Reaching 2000 on this bead alone is not the contract.

**Architecture:** Claude Code harness assembles `skill_listing` by concatenating each enabled plugin's `SKILL.md` frontmatter `description:` field; assembles `deferred_tools_delta` by listing MCP tool names from enabled-plugin manifests. Two levers: (a) shorten the `description:` fields we own (Sylveste-monorepo `interverse/*/skills/*/SKILL.md`, 78 skills, 11.5 KB of descriptions), and (b) disable external plugins whose skills/tools are cold-in-usage (measured via `tool_selection_events` + `agent_runs`).

**Tech Stack:** bash · python3 · PyYAML · git · cass analytics (read-only) · Claude Code plugin manifest format.

**Prior Learnings:**
- `docs/research/2026-04-21-sylveste-cost-breakdown.md` (sylveste-8lsf) — baseline numbers, target metrics, and the measured tool/MCP call counts we'll cross-reference for the cold-plugin list.
- MEMORY.md feedback_long_term_quality_default.md — prefer quality trims (preserve trigger semantics) over aggressive byte-shaving.
- MEMORY.md beads-troubleshooting.md — avoid `bd edit`; all bd updates go via `--title/--description/--notes/--design`.

- After /clear in a fresh session, total preamble is ≥2000 tokens smaller than the baseline captured on 2026-04-21.
- Every skill that was discoverable before this work is still discoverable (i.e., skill trigger descriptions preserved the nouns/verbs the harness matches against).
- MCP tools used in the last 30 days (>5 calls in `tool_selection_events`) still load or remain in the deferred set — none are silently removed.

**Key Links** (connections where breakage cascades):

#!/usr/bin/env bash
# Emits JSON with preamble-component byte counts from a session jsonl.
# Defaults to the newest jsonl in the project dir; override with --session-id
# (explicit ID avoids the ls -t race when the current session is still writing).
set -euo pipefail
project_dir="${HOME}/.claude/projects/-home-mk-projects-Sylveste"
session_id=""
for arg in "$@"; do
  case "$arg" in
    --session-id=*) session_id="${arg#--session-id=}" ;;
  esac
done

if [[ -n "$session_id" ]]; then
  target="$project_dir/$session_id.jsonl"
  [[ -f "$target" ]] || { echo "{\"error\":\"no jsonl for $session_id\"}"; exit 1; }
else
  target=$(ls -t "$project_dir"/*.jsonl 2>/dev/null | head -1)
  [[ -z "$target" ]] && { echo '{"error":"no session jsonl"}'; exit 1; }
fi

python3 - "$target" <<'PY'
import json, sys
path = sys.argv[1]
buckets = {
  "skill_listing_bytes": 0,
  "deferred_tools_delta_bytes": 0,
  "mcp_instructions_bytes": 0,
  "sessionstart_bytes": 0,
  "total_preamble_bytes": 0,
  "sampled_session_id": path.split("/")[-1].replace(".jsonl",""),
  "attachment_types_seen": [],
}
seen_types = []
with open(path) as f:
    for line in f:
        try: m = json.loads(line)
        except:
            # Still count for total_preamble_bytes if we haven't hit an assistant turn yet
            buckets["total_preamble_bytes"] += len(line)
            continue
        if m.get("type") == "assistant":
            # First assistant turn ends preamble. Do NOT use line-number guard —
            # sessions with many async hook responses can push skill_listing past
            # any fixed cutoff, producing a silent zero-byte mismeasurement.
            break
        buckets["total_preamble_bytes"] += len(line)
        att = m.get("attachment") or (m.get("message") or {}).get("attachment") or {}
        if not isinstance(att, dict):
            msg_att = m.get("message", {}).get("attachment")
            if isinstance(msg_att, dict): att = msg_att
        t = att.get("type", "")
        if t and t not in seen_types: seen_types.append(t)
        if t == "skill_listing":
            buckets["skill_listing_bytes"] += len(line)
        elif t == "deferred_tools_delta":
            buckets["deferred_tools_delta_bytes"] += len(line)
        elif t == "mcp_instructions_delta":
            buckets["mcp_instructions_bytes"] += len(line)
        elif t == "hook_success" and att.get("hookEvent") == "SessionStart":
            buckets["sessionstart_bytes"] += len(line)

buckets["attachment_types_seen"] = seen_types
# Schema validation: if skill_listing wasn't observed at all, this jsonl is
# probably mid-session or malformed — emit an explicit warning rather than silently 0.
if buckets["skill_listing_bytes"] == 0:
    buckets["warning"] = "skill_listing attachment not found — check --session-id or re-run after /clear"
print(json.dumps(buckets, indent=2))
PY
```

#!/usr/bin/env python3
"""Audit per-plugin skill_listing byte contribution, cross-reference usage."""
import argparse, glob, json, os, sqlite3, sys
from collections import defaultdict

def parse_frontmatter(text):
    if not text.startswith('---'): return {}
    end = text.find('---', 3)
    if end < 0: return {}
    out = {}
    for ln in text[3:end].splitlines():
        if ':' in ln:
            k, _, v = ln.partition(':')
            out[k.strip()] = v.strip()
    return out

def main():
    ap = argparse.ArgumentParser()
    ap.add_argument('--plugins-root', default=os.path.expanduser('~/.claude/plugins'))
    ap.add_argument('--db', default=os.path.expanduser('~/.claude/interstat/metrics.db'))
    ap.add_argument('--since-days', type=int, default=30)
    ap.add_argument('--out', default='-')
    args = ap.parse_args()

    # Collect one SKILL.md per plugin (dedup by plugin name, prefer shortest path)
    per_plugin = defaultdict(lambda: {"skills": 0, "desc_bytes": 0, "examples": []})
    seen = set()
    for p in sorted(glob.glob(f"{args.plugins_root}/**/SKILL.md", recursive=True), key=len):
        parts = p.split('/')
        try:
            plugin = parts[parts.index('skills') - 1]
        except (ValueError, IndexError):
            plugin = 'unknown'
        key = (plugin, os.path.basename(os.path.dirname(p)))
        if key in seen: continue
        seen.add(key)
        try:
            fm = parse_frontmatter(open(p).read())
        except Exception:
            continue
        desc = fm.get('description', '').strip().strip('"\'')
        per_plugin[plugin]["skills"] += 1
        per_plugin[plugin]["desc_bytes"] += len(desc)
        if len(per_plugin[plugin]["examples"]) < 3:
            per_plugin[plugin]["examples"].append(os.path.basename(os.path.dirname(p)))

    # Usage: MCP calls + Skill invocations per plugin, last N days
    usage = defaultdict(int)
    if os.path.exists(args.db):
        c = sqlite3.connect(args.db)
        cutoff_date = f"date('now','-{args.since_days} days')"
        for tool, calls in c.execute(f"""
            SELECT tool_name, COUNT(*) FROM tool_selection_events
            WHERE timestamp >= {cutoff_date}
            GROUP BY tool_name
        """):
            if not tool: continue
            if tool.startswith('mcp__plugin_'):
                plugin = tool.split('__')[1].replace('plugin_','',1).split('_')[0]
                usage[plugin] += calls
            elif tool == 'Skill':
                usage['_skill_invocations'] += calls

    rows = []
    for plugin, info in per_plugin.items():
        rows.append({
            "plugin": plugin,
            "skills": info["skills"],
            "desc_bytes": info["desc_bytes"],
            "mcp_calls_30d": usage.get(plugin, 0),
            "examples": info["examples"],
            "cold": usage.get(plugin, 0) < 5 and info["desc_bytes"] > 500,
        })
    rows.sort(key=lambda r: -r["desc_bytes"])
    out = json.dumps(rows, indent=2)
    if args.out == '-':
        print(out)
    else:
        with open(args.out, 'w') as f: f.write(out)

if __name__ == '__main__':
    main()
```

**Step 2: Define trim policy** (document at top of edit session)

Preserve: trigger nouns/verbs (the words the harness matches against), disambiguating clauses ("NOT for X").
Cut: "This skill should be used when..." boilerplate prefix; repeated trigger lists (>4 example phrases); explanatory tails ("Provides comprehensive guidance for..."); redundant "Use when" / "Trigger when" pairs.
Target: ≤120 chars for complex skills, ≤80 for simple ones, minimum 40 chars (must still contain at least one trigger verb + one trigger noun).

**Step 2.5: Extract trigger vocabulary per skill** (gate before any edit)

The harness indexes skill matches on the description field. Shortening below the trigger threshold silently breaks discovery. Before editing each file, compute the original's trigger vocabulary and record it:

```python
# scripts/perf/extract-trigger-vocab.py
import sys, re, json
text = open(sys.argv[1]).read()
end = text.find('---', 3)
desc = ""
for ln in text[3:end].splitlines():
    if ln.lstrip().startswith('description:'):
        desc = ln.split(':',1)[1].strip().strip('"\'')
        break
# Trigger vocab = content nouns/verbs from "Use when X", "Trigger when X", and quoted phrases
vocab = set()
# Quoted phrases (e.g., "create a skill")
for m in re.finditer(r'"([^"]+)"', desc):
    for w in m.group(1).split():
        if len(w) > 3: vocab.add(w.lower().strip(',."-:'))
# Use when / Trigger when / When X clauses
for m in re.finditer(r'(?:use when|trigger when|when the user)[\s:]+([^.]+)', desc, re.I):
    for w in m.group(1).split():
        if len(w) > 3: vocab.add(w.lower().strip(',."-:'))
# All content words length > 5 (rare nouns most likely to be distinguishing)
for w in desc.split():
    w2 = w.lower().strip(',."-:')
    if len(w2) > 5 and w2 not in {'should','skill','where','while','across'}:
        vocab.add(w2)
print(json.dumps(sorted(vocab)))
```

Capture pre-edit vocab into `/tmp/trigger-vocab-<skill>.json` before each edit.

import yaml, glob, sys, json, os, re
bad=[]
for p in glob.glob('interverse/*/skills/*/SKILL.md'):
    t=open(p).read()
    end=t.find('---',3)
