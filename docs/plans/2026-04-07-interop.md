# Plan: interop — Unified Integration Fabric

## Feature Ordering

```
Phase 1: F1 (core daemon + event bus)
Phase 2: F2 (beads) + F5 (filesystem) — parallel, both depend only on F1
Phase 3: F4 (Notion) + F3-partial (GitHub Issues/PRs, no file sync) — parallel
Phase 4: F3-full (GitHub repo file sync — requires F4 + F5)
Phase 5: F6 (MCP server + plugin)
Phase 6: F7 (interkasten migration — requires all adapters running)
```

Each phase gate: all tasks in phase pass tests before next phase begins. Phases 2 and 3 have internal parallelism.

### Task 1.2: Event type + Adapter interface

Optional: `HealthChecker` (`Health() HealthStatus`), `StateSyncer` (`FullSync(ctx) error`)
3. `errors.go` — typed errors: `ErrMalformed` (discard), `ErrTransient` (retry), `ErrConflict` (escalate to SyncJournal), `ErrShuttingDown` (requeue), `ErrUnresolvableTarget` (dead-letter, not silent drop)
**Test:** Compile check; mock adapter implements interface

### Task 1.4: EventBus with per-entity serialization + Emit validation

**Files:** `internal/bus/bus.go`, `internal/bus/bus_test.go`
**Steps:**
1. `EventBus` struct: registered adapters, per-entity dispatch channels (`sync.Map` keyed by `EntityKey` → `chan Event`, created on demand, GC'd after 60s idle — `sync.Map` required: concurrent adapter goroutines create channels simultaneously, plain map races. P0 fix from plan review), three-lane priority select (`express`/`urgent`/`routine`)
2. `Emit(e Event) error` — validates before enqueueing: non-empty EntityID, non-nil Payload, valid Type, non-empty SourceAdapter. Invalid → logged + dropped (contamination barrier — fixes P0-6 from flux-review)
3. `Register(a Adapter)` / `Unregister(a AdapterID)`
4. Dispatch loop: read from adapter Emit channels → classify priority → route to per-entity channel → fan out to other adapters' HandleEvent. Skip routing back to source adapter (cycle prevention via SourceAdapter check + hop count)
5. `Requeue(e Event)` — for panic recovery in adapter goroutine pools. **Retry limit:** max 3 requeues per event (tracked via `Event.RetryCount`), then dead-letter to SyncJournal with `ErrMaxRetries` (P1 fix: prevent infinite requeue loops)
**Test:** 2 mock adapters, emit from A, verify B receives; emit from B, verify A receives; emit invalid event, verify rejected; emit same EntityKey from A and B simultaneously, verify serialized (not interleaved)

### Task 1.5: CollisionWindow

**Files:** `internal/bus/collision.go`, `internal/bus/collision_test.go`
**Steps:**
1. `CollisionWindow` struct: `map[EntityKey]pendingEvent` protected by `sync.Mutex` (P0 fix: concurrent reads/writes from dispatch goroutines race without mutex) with configurable TTL (default 5s)
2. Before dispatch, check if opposing-source event for same EntityKey exists in window
3. On collision: route pair to `ConflictResolver.ResolvePair(a, b Event)` — write `ConflictRecord` to SyncJournal with `status=pending` (fixes P1-ACQ-01: externally observable via `/conflicts`)
4. No collision: add to window, dispatch after TTL expires or next tick
5. Periodic cleanup of expired entries
**Test:** Two opposing events within TTL → conflict detected; two events outside TTL → both dispatched; same-source events → no collision

### Task 1.6: SyncJournal (SQLite)

**Files:** `internal/journal/journal.go`, `internal/journal/journal_test.go`, `internal/journal/schema.sql`
**Steps:**
1. SQLite table: `sync_operations` (id, event_id, entity_id, source_adapter, dest_adapter, status [begun/completed/failed/conflict], started_at, completed_at, error_msg, conflict_record JSON)
2. Methods: `Begin(eventID, entityID, src, dst) error`, `Complete(eventID) error`, `MarkFailed(eventID, err) error`, `ResolveConflict(entityID, strategy, val1, val2, winner) ConflictRecord`
3. `ListPending() []SyncOp` — for retry on startup
4. `ListConflicts(status string) []ConflictRecord` — for MCP query
5. WAL mode enabled, write-ahead logging
**Test:** Begin+Complete round-trip; Begin+MarkFailed round-trip; ListPending after crash (Begin without Complete); restart persistence (close DB, reopen, verify data)

### Task 1.7: AncestorStore (SQLite)

**Files:** `internal/ancestor/store.go`, `internal/ancestor/store_test.go`
**Steps:**
1. SQLite table: `ancestors` (entity_id PRIMARY KEY, content_hash, content BLOB, adapter_pair, updated_at)
2. `Get(entityID string) (Ancestor, error)`, `Put(entityID string, a Ancestor) error`, `Delete(entityID string) error`
3. Indexed by entity_id for fast lookup
4. **Restart persistence test** (fixes P0-1): store N ancestors, close store, reopen, verify all N retrievable with identical content
**Test:** Put+Get round-trip; restart persistence; concurrent Put from different goroutines (SQLite serialized writes)

### Task 1.8: ConflictResolver with per-adapter-pair policies

**Files:** `internal/conflict/resolver.go`, `internal/conflict/resolver_test.go`, `internal/conflict/policy.go`
**Steps:**
1. `ConflictPolicy` per adapter pair: `beads+github` → beads authoritative for status, GitHub for PR state; `beads+notion` → beads for status, Notion for prose content; `notion+filesystem` → three-way merge with AncestorStore; configurable in YAML (fixes P1-BSC-04)
2. `ResolvePair(a, b Event) (Event, ConflictRecord)` — applies policy, writes to SyncJournal, returns winning event
3. `ResolveThreeWay(base, left, right) (merged, ConflictRecord, error)` — three-way merge using AncestorStore. Returns `ErrUnresolvable` when field-level divergence cannot be auto-merged → dead-letter to SyncJournal (P1 fix: no dead-letter path). Field-level comparison even on convergent state (fixes supply-chain P0-5: same close-state, different close-notes → dead-letter)
4. LWW uses `interop receive-time` monotonic clock, NOT external timestamps (fixes P1-BSC-02)
**Test:** Per-pair policy selection; three-way merge produces correct output; field-level conflict on convergent state detected; LWW uses interop timestamp

### Task 1.9: Daemon lifecycle + graceful shutdown

**Files:** `internal/daemon/daemon.go`, `internal/daemon/daemon_test.go`
**Steps:**
1. `Daemon` struct: holds EventBus, SyncJournal, AncestorStore, registered adapters, HTTP server (health + webhooks), MCP server
2. `Run(ctx context.Context) error` — start all components, block until context cancelled
3. Graceful shutdown via `signal.NotifyContext(ctx, syscall.SIGTERM, syscall.SIGINT)`:
   - Close HTTP listener (stop new webhooks)
   - Cancel adapter contexts
   - Wait for in-flight events to drain (configurable timeout, default 30s)
   - Flush SyncJournal and AncestorStore
   - Close MCP server
   - Exit
4. Health endpoints: `/health` (liveness — process alive), `/ready` (readiness — all adapters started, each adapter's `Ready() <-chan struct{}` closed)
5. Structured JSON logging via `slog` with adapter context fields
6. MCP/webhook lifecycle coupling: if either server crashes, trigger graceful shutdown of both (fixes flux-review daemon insight)
**Test:** SIGTERM triggers orderly shutdown; in-flight events complete before exit (verify via SyncJournal: no `begun` entries without `completed` after shutdown — P1 fix: drain test must verify events actually complete); health returns 200 after start; ready returns 503 before all adapters ready, 200 after

1. Implements `Adapter` interface. `Start()` spawns polling goroutine

6. Failure writes `ErrTransient` to SyncJournal (fixes P1-ACQ-02)
7. Passes adapter compliance test suite
**Test:** Mock bd CLI output → correct events emitted; incoming event → correct bd command executed; dedup: same state → no event; compliance suite passes
