#### Task 2: Implement Render function
- [x] Define `RenderResult` struct: `Prompt string`, `Included []string`, `Excluded []string`, `ExcludedStable []string`, `TotalTokens int`, `StableTokens int`
- [x] Implement `Render(elements []Element, budget int, opts ...Option) RenderResult`
- [x] Apply defaults: separator `"\n\n"`, tokenizer `CharHeuristic{Ratio: 4}`, no active phase
- [x] Handle edge cases first:
  - nil/empty elements → empty result
  - Filter out empty-Content elements first (before any budget logic)
  - budget <= 0 → remaining (non-empty) elements go to Excluded/ExcludedStable, empty Prompt
- [x] Compute effective priority: `Priority + PhaseBoost[activePhase]` (0 if no active phase or no matching key; nil map is safe)
- [x] Partition elements into stable and dynamic slices
- [x] Sort each partition: by effective priority descending, then by Name ascending for ties
- [x] Greedy pack: iterate stable then dynamic. Track `includedCount := 0`. For each element:
  - `tokenCost := tokenizer.Count(element.Content)`
  - `separatorCost := 0` — only add separator cost if `includedCount > 0`: `separatorCost = max(1, tokenizer.Count(separator))`
  - If `tokenCost + separatorCost <= remainingBudget`, include it: `remainingBudget -= tokenCost + separatorCost; includedCount++`
  - Otherwise add name to `Excluded` (or `ExcludedStable` for stable elements)
- [x] If any stable element is excluded, set `StableTokens = 0`
- [x] Otherwise, compute `StableTokens` = `tokenizer.Count(strings.Join(stableContents, separator))` on the concatenated stable substring (not accumulated per-element — avoids integer-division non-additivity)
- [x] Concatenate all included elements with separator between adjacent elements (no leading/trailing separator)
- [x] Set `TotalTokens = tokenizer.Count(result.Prompt)` — single canonical path, not accumulated
- [x] `Excluded` list is naturally ordered by effective priority descending (loop iterates in priority order, so appending in loop order produces this ordering)
- **N-1 separators:** N included elements produce N-1 separators. separatorCost is 0 for the first included element.
- **Critical:** First-element separator cost MUST be 0. A single element whose token cost exactly equals the budget must be included.
