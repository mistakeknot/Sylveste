# AI Coding CLI/TUI Landscape Research

**Date:** 2026-03-11
**Purpose:** Survey of notable AI coding agent CLIs and their TUI/UX features, for informing Skaffen and Clavain design decisions.

---

## 1. Aider

**URL:** https://aider.chat | **Repo:** https://github.com/Aider-AI/aider
**Language:** Python | **Open source:** Yes (Apache 2.0)
**Stars:** ~30k+

### Git Integration (the defining feature)

Git is not a feature in Aider — it is the foundation. Every AI edit becomes a git commit with a descriptive commit message generated by a "weak model." Aider takes special care with files that already have uncommitted changes: it first commits preexisting changes with a separate descriptive message, keeping your edits and AI edits cleanly separated.

- Commit messages follow Conventional Commits by default; customizable via `--commit-prompt`
- Pre-commit hooks supported via `--git-commit-verify` (skipped by default with `--no-verify`)
- `/diff` command shows diffs of all changes from a request, including lint and test fixes

### Architect/Editor Model Split

Separates reasoning from editing into two model passes:

1. **Architect model** (e.g., o1, o3) proposes changes in natural language — describes the solution however comes naturally
2. **Editor model** (e.g., GPT-4o, Sonnet) translates the proposal into specific file edits using the configured edit format

This is configured per-session and lets you pair a strong reasoning model with a fast/cheap editing model.

### Chat Modes

- **Code mode** (default): Direct file editing
- **Architect mode**: Two-model proposal + edit pipeline
- **Ask mode**: Questions without edits
- **Help mode**: Aider documentation queries

### Voice Mode

`/voice` command captures audio via PortAudio, transcribes with OpenAI Whisper (can run locally for privacy), and feeds the transcription as a chat message. You can fluidly switch between voice and text within the same session.

### Repository Map

Uses tree-sitter to parse source files and extract definitions/references, then builds a NetworkX graph of file dependencies. PageRank ranks code elements by relevance. The map is token-budgeted (default 1k tokens via `--map-tokens`) and dynamically resized based on chat state.

The repo map includes the most important identifiers — the ones most often referenced by other code — giving the LLM understanding of the overall codebase structure without consuming the entire context window.

### Edit Formats

Aider uses different "edit formats" optimized per model:

| Format | How it works | Default for |
|--------|-------------|-------------|
| **diff** (search/replace) | LLM emits SEARCH/REPLACE blocks | GPT-4o |
| **udiff** (unified diff) | Simplified unified diff format; only changed regions | GPT-4 Turbo (reduced laziness 3x) |
| **whole** | LLM returns full updated file | GPT-3.5 |

The edit format is auto-selected per model but overridable with `--edit-format`.

### Watch Mode

`--watch-files` monitors your repo for AI comments in any IDE/editor:

- `// AI!` triggers code changes
- `// AI?` triggers question answering
- Comments are sent to the LLM with the repo map and file context
- Enables a "write comments in your IDE, get edits from aider" workflow

### Linting & Testing Integration

- Auto-lint after every change (enabled by default)
- Configurable lint commands per language
- Auto-test support with configurable test commands
- On failure, aider feeds lint/test output back to the LLM for self-correction

### Configuration

Three layers:
1. `.aider.conf.yml` — YAML config (home dir or git root)
2. `.env` file — API keys and settings (home dir, git root, or cwd; later files override)
3. `.aider.model.settings.yml` — Advanced per-model LiteLLM parameters

All options also available as CLI flags and `AIDER_xxx` environment variables.

### Unique/Notable

- **Multi-file editing** with understanding of cross-file dependencies via repo map
- **Image and web page context** — add screenshots or URLs to the chat for visual context
- **Benchmarking culture** — Aider maintains public benchmarks comparing edit formats and models; 52.7% combined score with 257s completion, 126k tokens (mid-Feb 2026)
- **Minimal TUI** — deliberately simple prompt-based interface; no split panes or rich TUI. Prioritizes git-first workflow over visual polish

## 2. OpenCode / Crush

**URL:** https://opencode.ai | **Repo:** https://github.com/opencode-ai/opencode
**Language:** Go (originally), now TypeScript client + Zig rendering backend (OpenTUI)
**Open source:** Yes
**Note:** Originally "OpenCode", rebranded to "Crush" in 2026 under Charm team stewardship.

### TUI (the defining feature)

The most sophisticated TUI of any CLI coding agent. Built on OpenTUI (custom framework: TypeScript API + native Zig rendering backend). Not a raw CLI — closer to a terminal IDE:

- **Split panes**: sidebar shows session metadata, context usage, MCP server status, LSP status, file changes, TODOs
- **Themes**: configurable via `tui.json`, multiple built-in themes
- **Diff rendering**: `diff_style` setting — "auto" adapts to terminal width, "stacked" for single-column. Syntax-highlighted diffs rendered inline
- **Scrollable message list** with proper buffer management (resize without breaking layout)
- **Vim-like editor** for text input

### LSP Integration

Unique among CLI agents. Connects to Language Server Protocol servers to give the AI:
- Type information and symbol definitions
- Real-time diagnostics (errors, warnings)
- Reduces hallucinated types in TypeScript, Rust, Go, etc.

### Session Management

- Multiple concurrent sessions with context preserved
- Mid-session model switching (change LLMs while preserving conversation context)
- SQLite-backed persistent storage for conversations
- Non-interactive mode: pass prompt as CLI argument for scripting/automation

### Tool Permissions

Granular tool permissions with customizable commit attribution.

### Provider Support

75+ LLM providers including Claude, GPT-5, Gemini, local models via Ollama, AWS Bedrock, Groq, Azure OpenAI, OpenRouter.

### Cross-Platform

Broadest platform support: macOS, Linux, Windows, Android, FreeBSD, OpenBSD, NetBSD.

### Custom Commands

User-defined commands stored as Markdown files — quickly send predefined prompts.

## 3. Goose (Block)

**URL:** https://block.github.io/goose | **Repo:** https://github.com/block/goose
**Language:** Rust (core agent), Electron (desktop)
**Open source:** Yes
**Stars:** ~29k+, 368+ contributors

### Architecture

Modular Rust core shared between CLI and Electron desktop app. Block co-designed MCP with Anthropic, so Goose has deep MCP integration (3,000+ tools via MCP).

### Tool Permissions & Approval Modes

Three permission modes:

| Mode | Behavior |
|------|----------|
| **Manual** | Every tool call requires explicit approval |
| **Smart** | Auto-allows read-only operations; requires approval for state-changing actions |
| **Auto** | No approval required |

Per-tool permission overrides on top of the mode. Future plans include secure auth flows where the agent cannot fake approval.

### Recipes

YAML files bundling instructions, required extensions, parameters, and retry logic into shareable configurations. Recipes can be scheduled with cron syntax:

```
goose schedule add --schedule-id daily-report --cron "0 0 9 * * *" --recipe-source ./recipes/daily-report.yaml
```

### Session Management

- SQLite-backed (sessions.db) since v1.10.0, migrated from individual .jsonl files
- `--fork`: duplicate session with copied history
- `--resume`: continue previous sessions
- Named sessions with `--name` or `--session-id`

### Subagents

`Agent::new()` creates isolated execution contexts. Each subagent has its own agent instance with separate state.

### IDE Integration

Works with VS Code, Cursor, Windsurf, and JetBrains IDEs through Agent Client Protocol (ACP).

### Unique/Notable

- **Recipe system** for repeatable automated workflows with scheduling
- **Desktop + CLI** sharing the same Rust core
- **MCP-first** extensibility (Block co-authored MCP)
- Steeper learning curve than competitors due to recipe/extension system

## 4. Cline CLI 2.0

**URL:** https://cline.bot/cli | **Repo:** https://github.com/cline/cline
**Language:** TypeScript
**Open source:** Yes (Apache 2.0)

### Plan and Act Modes

The defining UX pattern. **Tab** toggles between:

- **Plan mode**: Agent reasons through the codebase, shows strategy. Different model can be configured (e.g., reasoning model for planning)
- **Act mode**: Agent executes changes

Each mode can use independent AI models — reasoning model for planning, fast model for execution.

### Approval Controls

- **Per-step approval**: Review each tool call before execution
- **Shift+Tab**: Toggle auto-approve mode (removes all approval prompts)
- **`-y` flag**: Full autonomy from the start

### TUI Features

- Markdown rendered inline with streaming responses
- Mode toggle via Tab (mirrors VS Code extension interaction patterns)
- Slash commands: `/history` (resume previous task), `/settings` (change config mid-session)
- `/history` for session resumption

### Automation & Scripting

- Full stdin/stdout support — pipe content in, chain commands
- `--json` flag streams structured NDJSON output (system init, deltas, tool calls, result)
- Usable in GitHub Actions, GitLab CI, Jenkins, cron jobs
- Each CLI instance is fully isolated (own state, conversation, model config)

### Unique/Notable

- **VS Code extension parity**: CLI mirrors the same interaction patterns as the VS Code extension
- **Plan/Act separation** with per-mode model configuration is distinctive
- **JSON streaming mode** for programmatic consumption

## 5. Continue CLI (cn)

**URL:** https://docs.continue.dev/cli | **Repo:** https://github.com/continuedev/continue
**Language:** TypeScript (Node.js)
**Open source:** Yes
**Status:** Alpha (Beta coming soon)

### Two Modes

- **Interactive**: `cn` — full terminal UI with approval prompts
- **Headless**: `cn -p "prompt"` — outputs only final response; Unix-philosophy scripting

### Permission System

Progressive trust model:
1. Starts with minimal permissions
2. As you approve tool calls, policies saved to `~/.continue/permissions.yaml`
3. "Continue + don't ask again" in TUI permanently allows a tool
4. CLI flags for session-scoped overrides: `--allow`, `--ask`, `--exclude`

### Configuration

Uses the same `config.yaml` as the Continue IDE extension. Compatible with Continue Mission Control for centralized config.

### Async Cloud Agents

`cn` supports async agents that work concurrently in the cloud — multiple AI-driven operations without blocking your main workflow.

### Unique/Notable

- **Shared config** between IDE extension and CLI — one config for both
- **Permission YAML** with progressive trust is a clean pattern
- **Headless mode** designed explicitly for Unix pipeline integration

## 6. Factory Droid CLI

**URL:** https://factory.ai/product/cli | **Docs:** https://docs.factory.ai
**Language:** Proprietary
**Open source:** No (commercial; community extensions on GitHub)

### Autonomy Levels

The most granular autonomy tiering of any agent. Cycle through levels with **Shift+Tab**:

| Level | Allowed |
|-------|---------|
| **Default** | Read-only operations |
| **Low** | Safe edits, reconnaissance |
| **Medium** | Development work, reversible dependency installs, build scripts |
| **High** | Migrations, integration tests, end-to-end scripts |

Commands auto-execute only when risk rating is at or below chosen autonomy level. Dangerous commands and command substitutions always require explicit approval regardless of level.

### Spec Mode

Research-and-plan without making changes. Droid creates a specification document, saves approved specs as markdown in `.factory/docs/`. Users review the plan before execution begins.

### Diff & Change Review

Native diff viewing in the terminal. Auto-Run shows streamed output and highlights every file change regardless of autonomy level.

### Slash Commands

Supports custom slash commands:
- Static markdown templates (predefined prompts)
- Executable scripts that run in your environment and return results to chat
- Example: `/run-test` triggers test suite, returns results to agent

### Context Tagging

Annotate files/sections for the agent's attention. Context-confidence compression with anchor points for large codebases.

### Multi-Interface

Available across CLI, Web, Slack, Linear, IDE (VS Code, Cursor, Windsurf, JetBrains), and custom scripts.

### Unique/Notable

- **Risk-rated autonomy levels** are the most sophisticated approval system surveyed
- **Spec mode** for plan-then-execute workflow
- **Custom slash commands** as both templates and executable scripts
- Enterprise-focused with multi-interface deployment

## 7. Cursor CLI

**URL:** https://cursor.com/cli | **Docs:** https://cursor.com/docs/cli
**Language:** Proprietary
**Open source:** No (commercial)

### Modes

- **Interactive**: Real-time collaboration with AI, proposed changes shown as navigable diffs
- **Headless/Print**: `cursor-agent -p -m gpt-5 "prompt"` — for scripts and CI/CD
  - `--output-format json` for single result JSON
  - `--output-format stream-json` for NDJSON events (system init, deltas, tool calls, result)

### Agent Modes

Plan and Ask modes (added Jan 2026). Agent can execute shell commands with sandboxing, preserved history, and native terminal integration.

### Context Selection

`@` mentions to include specific files or folders. MCP support and rule integration.

### Session Management

- `cursor-agent ls` — list sessions
- `cursor-agent resume` / `--resume <id>` — resume threads
- Headless sessions cannot be resumed (limitation)

### Diff Display

Interactive mode shows proposed changes as navigable diffs with keyboard shortcuts to accept/reject individual changes.

### Unique/Notable

- **Tight integration with Cursor IDE** — CLI extends the same agent to terminal
- **Cloud handoff** for background tasks
- **One-click MCP auth**

## 8. Warp AI

**URL:** https://www.warp.dev | **Docs:** https://docs.warp.dev
**Language:** Rust (terminal emulator)
**Open source:** Partially (Warp terminal is proprietary; some components open)

### Not a CLI Agent — an AI-Native Terminal Emulator

Warp is fundamentally different: it's a terminal emulator with AI embedded, not a standalone agent.

### Agent Mode

Two sub-modes:
- **Pair**: AI assists alongside you as you work — suggests commands, explains output
- **Dispatch**: AI works independently on complex tasks, creating and executing plans

### Approval & Control

- Approve commands one-at-a-time
- Turn on auto-approval for similar commands per session
- Refine the request
- Take over manually

### Oz Orchestration Platform (Feb 2026)

Cloud agent orchestration:
- CLI, API, and SDK for building autonomous cloud agents
- Cloud environments with full terminal use and computer use (screen verification)
- Built-in scheduler
- Can orchestrate multiple agents: Warp's own agent, Claude Code, Codex, Gemini CLI — all from the terminal

### Warp Drive

Team knowledge sharing — shared commands, workflows, and documentation.

### Unique/Notable

- **Terminal-first**: AI is embedded in the terminal itself, not a separate CLI tool
- **Oz** for orchestrating multiple different coding agents from one terminal
- **Computer Use** capability — agents can verify changes visually
- **120fps rendering** (Rust-based terminal emulator)

## 9. Zed AI

**URL:** https://zed.dev/docs/ai | **Repo:** https://github.com/zed-industries/zed
**Language:** Rust (editor)
**Open source:** Yes (GPL v3 / AGPL v3)

### Not a CLI — an AI-Native Editor with Agent Panel

Zed is a GPU-accelerated code editor (120fps) with built-in AI agent capabilities.

### Agent Panel

- Opens with `cmd-?` / `ctrl-?`
- Agents can read files, edit code, run terminal commands, search web, access diagnostics
- Watch AI edit multiple files simultaneously at 120fps in real-time
- Extensible with MCP servers

### Agent Client Protocol (ACP)

Open standard (co-developed by Zed) for connecting any editor to any agent:
- Run Claude Code, Gemini CLI, Codex, Factory Droid, and others directly in Zed's agent panel
- JSON-RPC 2.0 over stdin/stdout
- Session history persisted across restarts

### Subagents

`spawn_agent` tool for parallel sub-tasks. Depth-limited: `MAX_SUBAGENT_DEPTH = 1` prevents recursive spawning.

### External Agent Support

Claude Agent, Gemini CLI, Codex, Factory Droid, Junie, Kimi CLI, Mistral Vibe, OpenCode — all via ACP with session history exposed.

### Unique/Notable

- **ACP** is a significant standardization effort (like MCP but for editor-agent communication)
- **120fps real-time multi-file editing visualization**
- **spawn_agent** for agent-initiated parallelism

## 10. Amp (Sourcegraph)

**URL:** https://ampcode.com | **Repo:** N/A (commercial, Sourcegraph)
**Language:** TypeScript (CLI via npm)
**Open source:** No (commercial, free tier available)

### Thread Management

Threads are the core abstraction:
- Persistent conversations synced to ampcode.com
- Resume threads across devices (laptop, server, phone)
- Colleague thread sharing
- Thread leaderboards for team collaboration

### Subagents

Can spawn subagents for parallel work:
- Each subagent has its own context window
- Uses tools independently (file editing, terminal commands)
- Reports back when done, keeping main thread clean
- Best for: multi-step tasks, operations with extensive output, parallel work across code areas

### Built-in Sub-agents

- **Oracle**: GPT-5-based tool for complex reasoning/analysis
- **Librarian**: Analyzes external libraries
- **Code Review Agent**: Automated code review
- **Painter**: Image generation tool

### CLI Features

Full terminal UI (evolved beyond raw CLI). Install via `npm install -g @sourcegraph/amp`. Supports MCP for extensibility.

### Context

Uses Claude Opus 4.6 for most tasks with up to 200k tokens of context. AGENTS.md support for project/folder/global context.

### Unique/Notable

- **Thread sync across devices** — start on laptop, finish on phone
- **Named sub-agents** (Oracle, Librarian) with specialized roles
- **Team features** — thread sharing, leaderboards

## 11. Gemini CLI (Google)

**URL:** https://github.com/google-gemini/gemini-cli
**Language:** TypeScript
**Open source:** Yes (Apache 2.0)

### Key Features

- **ReAct loop** with built-in tools and MCP support
- **1M token context window** — largest of any CLI agent
- **Google Search grounding** — agent can search the web to verify answers
- **Conversation checkpointing** — save and resume sessions exactly where you left off
- **Three auth tiers**: free personal (60 req/min, 1000/day), cloud, enterprise Vertex AI

### Built-in Tools

Google Search grounding, file operations, shell commands, web fetching.

### Unique/Notable

- **1M token context** is unmatched — can ingest entire large codebases
- **Free tier** is generous (1000 requests/day)
- **Search grounding** for fact-checking/verification during coding

## Comparative Summary

### Approval/Autonomy Models

| Tool | Approach |
|------|----------|
| **Factory Droid** | 4-tier risk-rated autonomy (default/low/medium/high) — most granular |
| **Goose** | 3-mode (manual/smart/auto) + per-tool overrides |
| **Continue** | Progressive trust with persistent YAML policies |
| **Cline** | Binary (per-step vs auto-approve via Shift+Tab) |
| **Cursor** | Per-command approval with sandboxing |
| **Warp** | Per-command with session auto-approve for similar commands |
| **Aider** | No approval system — trusts the LLM, relies on git for rollback |

### Diff/Change Display

| Tool | Approach |
|------|----------|
| **OpenCode/Crush** | Syntax-highlighted inline diffs, adaptive layout (auto/stacked) |
| **Aider** | `/diff` command shows all changes; edit format varies per model (search/replace, udiff, whole) |
| **Factory Droid** | Native diff viewing in terminal, streamed file change highlights |
| **Cursor** | Navigable diffs with per-change accept/reject keyboard shortcuts |
| **Cline** | Inline markdown rendering with streaming |

### Session/Context Management

| Tool | Approach |
|------|----------|
| **Amp** | Cloud-synced threads across devices, colleague sharing |
| **OpenCode/Crush** | SQLite-backed multi-session with mid-session model switching |
| **Goose** | SQLite sessions with fork/resume, named sessions |
| **Gemini CLI** | Conversation checkpointing with save/resume |
| **Cline** | Isolated instances, `/history` for resumption |
| **Aider** | Chat history in git repo; no formal session management |

### Codebase Understanding

| Tool | Approach |
|------|----------|
| **Aider** | Tree-sitter + PageRank repo map (token-budgeted) |
| **OpenCode/Crush** | LSP integration (type info, diagnostics, symbol definitions) |
| **Gemini CLI** | 1M token context window (brute-force ingest) |
| **Goose** | MCP-based tool access |

### TUI Sophistication (low to high)

1. **Aider** — Minimal prompt, git-first philosophy
2. **Continue CLI** — Clean TUI with approval prompts
3. **Cline CLI** — Markdown rendering, mode toggle, slash commands
4. **Amp** — Full terminal UI with thread management
5. **Cursor CLI** — Navigable diffs, agent modes
6. **Factory Droid** — Full-screen terminal, spec mode, context tags
7. **OpenCode/Crush** — Split panes, themes, LSP sidebar, vim editor — closest to terminal IDE

### Unique Patterns Worth Noting

- **Aider's watch mode** (AI comments in any editor) is an elegant IDE-agnostic integration
- **Factory's spec mode** (plan as markdown artifact before execution) is good for audit trails
- **Continue's progressive trust YAML** learns from user behavior
- **Goose's recipes** (YAML bundles with scheduling) enable repeatable automation
- **Zed's ACP** could become the standard for editor-agent communication (like MCP for tools)
- **Warp's Oz** for multi-agent orchestration from a single terminal
- **Amp's cross-device thread sync** enables mobile/server continuation
- **Aider's architect/editor split** optimizes model costs by pairing reasoning + editing models
- **OpenCode/Crush's LSP integration** gives the AI actual type-system awareness
