# Synthesis: Approaches for Auraken as Systems-Thinking Consultant

## Cross-Track Convergence

Findings independently surfaced by 2+ tracks at different semantic distances. Ranked by convergence score.

### 1. Calibration data is a curriculum, not a routing table (4/4 tracks)

The strongest signal across the entire review. Every track independently concluded that the Forge findings (454 near-miss pairs, 30 distinguishing questions, lens stacks, contraindications) should be operationalized as a **user-facing discrimination training system**, not just as internal lens-selection improvements.

- **Track A** (fd-cognitive-consultant-identity): DQs should build user skill, not just improve routing accuracy. Making calibration invisible creates a chatbot; making it pedagogical creates a consultant.
- **Track B** (fd-sommelier-sensory-discrimination-training): The near-miss pairs are a structured tasting curriculum. Present side-by-side contrast pairs to train the user's palate for problem structure.
- **Track C** (fd-wine-sommelier-client-development): Progressive challenge calibration over years — start with high-contrast pairs (Trilemma vs SBI), advance to near-identical pairs (A Model of Trust vs Trust Is a Long Game).
- **Track D** (fd-tang-civil-examination-keju): Keju commissioners embedded hidden structural tensions in exam questions. The DQs should force internalization of frameworks, not memorization. The xiucai → jinshi progression maps to a missing user-development ladder.

**Implication:** Auraken should present DQs *to the user* as part of the conversation, not use them silently for internal routing. The user's answer to "Is the difficulty that rules keep failing, or that three constraints compete?" is itself the cognitive work. Auraken's job is to calibrate which DQ is appropriate for this user at this stage.

### 2. Lens stacks are reference-frame inversions, not additive filters (3/4 tracks)

Three tracks independently rejected the "sequential application" model for lens stacks.

- **Track A** (fd-lens-stack-sequencing): Each lens in a stack doesn't add analysis — it redefines what the problem is about.
- **Track C** (fd-glassblowing-gather-sequence): Each gather transforms the glass irreversibly. You can't "undo" a reframe. The second lens operates on a transformed problem, not the original.
- **Track D** (fd-polynesian-etak-navigation): Etak navigators hold the canoe stationary while islands move. Each lens in a stack changes what is held fixed and what is variable. The problem moves, not the analysis.

**Implication:** Lens stacks should be designed as phase transitions, not checklists. The output of Phase 1 (e.g., "I'm avoiding") is the *input redefinition* for Phase 2 (e.g., "Now what specifically am I avoiding?"). Auraken should make these transitions explicit: "Your answer just changed what this problem is about. It started as a motivation question; now it's a boundary question."

### 3. User development ladder is missing (3/4 tracks)

- **Track B** (fd-od-consulting-framework-transfer): OD consultants track capacity-building milestones — when the client can apply the framework without the consultant.
- **Track C** (fd-wine-sommelier-client-development): Progressive palate development: obvious contrasts → subtle near-misses → unaided discrimination.
- **Track D** (fd-tang-civil-examination-keju): xiucai (basic) → jinshi (mastery) examination progression.

**Implication:** Auraken needs a discrimination difficulty ladder. Track the user's ability to answer DQs correctly without help. Start with high-contrast pairs (Trilemma vs SBI — 100% resolution rate in stress tests). Graduate to near-identical pairs (A Model of Trust vs Trust Is a Long Game — requires understanding ability/benevolence/integrity decomposition). Celebrate when users can self-select lenses.

### 4. Concealment depth should be calibrated per-user (3/4 tracks)

- **Track A** (fd-consulting-conversation-model + fd-cognitive-consultant-identity): The invisible-lens principle is right, but binary (invisible vs visible). A consultant modulates how much framework is showing.
- **Track C** (fd-organ-registration-voicing): Lens "voicing" changes when it's part of a stack — the same lens sounds different in context.
- **Track D** (fd-ethiopian-wax-and-gold-hermeneutics): The 8 Qene verse forms provide a taxonomy of concealment depth. The question isn't "invisible or visible?" but "at what depth should the gold be buried for THIS user?"

**Implication:** Replace the binary invisible/visible toggle with a graduated disclosure model. Early conversations: deep gold (Auraken asks questions that embody the lens without naming it). Intermediate: shallow gold (name the lens after the user has already applied it: "What you just did is called Chesterton's Fence"). Advanced: wax (teach the lens vocabulary directly, then ask the user to apply it).

### 5. Near-miss pairs have emergent combinatorial properties (2/4 tracks)

- **Track C** (fd-organ-registration-voicing): Two stops that clash as solos may produce rich composite insight as a registration. Near-miss pairs (hazardous in isolation) might be powerful *in combination*.
- **Track D** (fd-polynesian-etak-navigation): Reference frame shifts create new problem spaces that neither lens alone could reveal.

**Implication:** Don't just use near-miss data to *avoid* confusing pairs — explore whether deliberate combination of near-miss lenses produces insights neither alone can reach. A Model of Trust + Trust Is a Long Game might be more powerful than either, if the user understands why they're different. The contraindication becomes a co-application instruction.

### 6. Pacing and annealing between lens applications (2/4 tracks)

- **Track B** (fd-physical-therapy-movement-literacy): Between-session practice prescription — give the user something to notice between conversations.
- **Track C** (fd-glassblowing-gather-sequence): Annealing (controlled cooling) between gathers prevents thermal stress. Rushing between lenses causes cognitive fatigue.

**Implication:** Lens stacks need breathing room. After Phase 1 (e.g., Approach/Avoid), don't immediately jump to Phase 2 (Microboundaries). Let the user sit with the reframe. Ask: "What did you notice about your avoidance pattern since we last talked?" The between-session integration is where the real learning happens.

## Domain-Expert Insights (Track A)

- **Calibration-to-production transfer gap** (fd-calibration-to-production-transfer): The Forge findings come from synthetic daily_dilemmas. Real Auraken conversations are messier — longer context, emotional stakes, ambiguous problem statements. The RESOLVED/PARTIAL rates may not transfer directly. Need a live calibration loop: track DQ resolution rates in production conversations and compare to Forge baselines.
- **The "no lens" path** (fd-consulting-conversation-model): Expert consultants sometimes say "I don't have a framework for this — let's just talk it through." Auraken needs a graceful fallback when no lens fits, rather than forcing the nearest match. The contraindications help here — if all candidate lenses are contraindicated, that's a signal to drop framework application entirely.

## Parallel-Discipline Insights (Track B)

- **Judicial holdings structure** (fd-judicial-reasoning-distinguishing-doctrine): DQs should be structured like legal holdings: operative condition ("When the constraints are structural and immovable..."), rationale ("...because rejecting real constraints leads to magical thinking..."), and scope ("...use Trilemma, not Kobayashi Maru"). This structure makes DQs durable and revisable — you can update the rationale without changing the operative condition.
- **Compensatory pattern detection** (fd-physical-therapy-movement-literacy): When a user consistently avoids a problem type (always picks Trilemma, never engages Reckoning vs. Judgment), that's a compensatory pattern — they're routing around a weakness. Auraken should notice and gently challenge: "You've been great at identifying constraint trade-offs. I notice you haven't engaged with situations where the constraints themselves are questionable."

## Structural Insights (Track C)

- **Holistic vs. fragmented diagnosis** (fd-kampo-sho-differential-diagnosis): Kampo treats the person's constitution (sho), not the disease. The Match-Verify-Check pipeline risks fragmenting holistic judgment into a checklist. Auraken should maintain a "constitutional" read of the user's thinking patterns alongside the per-problem lens selection.
- **Productive discomfort vs. misapplication** (fd-kampo-sho-differential-diagnosis): The mengen reaction (temporary worsening from correct treatment) maps to cognitive discomfort from a correctly applied but challenging lens. Auraken needs failure signatures that distinguish "this lens is hard but right" from "this lens is wrong."

## Frontier Patterns (Track D)

- **Meserete subversion** (fd-ethiopian-wax-and-gold-hermeneutics): In Qene, the meserete (foundation verse) is what gets subverted — the poet establishes an expectation, then inverts it. Lens stacks should work the same way: the first lens establishes a frame, and the second lens *subverts* it. Not "Approach/Avoid then Microboundaries" but "the avoidance you identified? It's actually a boundary that's protecting something important." The subversion is where the insight lives.
- **Dead-reckoning fallback** (fd-polynesian-etak-navigation): When clouds obscure the stars and swells shift, Etak navigators fall back to dead reckoning — maintaining course from the last known position. When Auraken's lens selection is uncertain (no clear winner, no DQ resolves), the fallback should be the user's most recently internalized framework, not a random selection. "When you're not sure what lens to use, try applying [the one you used successfully last time]. What does it show you?"

## Synthesis Assessment

- **Overall design quality:** The vision and philosophy docs establish a strong consultant identity. The gap is between the *principles* (camera not engine, preserve cognitive struggle) and the *mechanisms* (how does calibration data operationalize these principles?). The Forge findings close this gap if treated as curriculum rather than routing.
- **Highest-leverage improvement:** Build the discrimination difficulty ladder. The 30 enriched near-miss pairs, ordered by difficulty (high-contrast → near-identical), are a ready-made progressive curriculum. Track user DQ resolution rates to advance them.
- **Surprising finding:** Near-miss pairs may be more powerful *in combination* than in isolation (organ registration convergence). The Forge data currently treats them as hazards to disambiguate; they may also be insights to compose.
- **Semantic distance value:** The outer tracks (C/D) contributed qualitatively different insights. Track A/B converged on "calibration as curriculum" (important but perhaps intuitable). Track C added temporal dynamics (irreversibility, annealing) and combinatorial emergence. Track D added the deepest reframe: lens stacks as reference-frame inversions (Etak), concealment-depth calibration (Wax-and-Gold), and internalization-vs-memorization testing (Keju). These would not have emerged from adjacent analysis alone.
