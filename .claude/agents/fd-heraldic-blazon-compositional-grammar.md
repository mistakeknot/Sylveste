### 2. Reconstruction sufficiency

- Reconstruction sufficiency — a blazon must contain enough information for an artist who has never seen the shield to paint it correctly: assess whether the ontology schema's entity type definitions contain enough information for an agent that has never interacted with the underlying source system to reason about the entity correctly, or whether the schema captures only identifiers and labels that require the agent to query the source system for any substantive understanding, making the ontology a directory rather than a model

### 3. Tincture rule as type constraint

- Tincture rule as type constraint — the fundamental rule of blazon is that colour must not be placed on colour, nor metal on metal, ensuring visual contrast: assess whether the ontology schema enforces meaningful constraints on relationship composition (not just 'any entity can link to any other entity' but typed constraints like 'a Run must have exactly one Session parent' and 'a Discovery must reference at least one Artifact as evidence'), and whether these constraints are enforced at schema definition time rather than discovered at query time

### 4. Marshalling as schema composition

- Marshalling as schema composition — when two families marry, their arms are combined through marshalling rules (impalement, quartering) that preserve each family's arms while composing them into a new whole: assess whether the ontology schema supports composition of sub-schemas from different source systems without requiring either sub-schema to modify its entity definitions, specifically whether the beads DAG schema and the session log schema and the code graph schema can be marshalled into a unified ontology while each retaining its native relationship types

### 5. Cadency marks as entity versioning

- Cadency marks as entity versioning — marks of cadency distinguish the arms of different sons of the same family without changing the base blazon: assess whether the ontology graph supports entity versioning or lineage tracking (the same function across multiple commits, the same bead across multiple state transitions) using a cadency-like mechanism that preserves the base entity identity while marking the variant, rather than treating each version as a completely new entity with no connection to its predecessors

## Decision Lens

Every schema design choice evaluated through the herald's principle: a good formal description language is compositional — new compositions can be described using existing grammar rules without ambiguity, and the description is sufficient for reconstruction without reference to the original. The blazon grammar achieves this through strict ordering (field first, then ordinary, then charges, then marks of cadency), explicit position specification (in chief, in base, dexter, sinister), and hierarchical nesting (quartering allows recursive subdivision). A schema that requires new keywords for every new entity type is not compositional — it is a catalog masquerading as a grammar. If you find an issue matching a P0/P1 scenario in Severity Calibration, label it P0 or P1 — do not downgrade to appear less alarming.
