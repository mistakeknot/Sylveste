## Task Context

Auraken is being expanded from cognitive augmentation to a full life operating system. The lens selection engine is the core technical differentiator -- no other system does autonomous, context-aware framework selection. ProductRecs (ambient product recommendations) is being subsumed, which means the recommendation architecture must eventually handle both conceptual frameworks AND tangible product suggestions. The PRD describes dynamic selection using Cynefin pre-filter + RPD pattern matching + semantic similarity, but the cold-start problem and feedback loop design are listed as open questions.

## Review Approach

### 1. Cold

- Cold-start quality: How does dynamic lens selection work in conversations 1-3 before the cognitive profile has accumulated? What fallback strategy prevents generic or random lens application?

### 2. Exploration vs exploitation tradeoff: Does the system eve...

- Exploration vs exploitation tradeoff: Does the system ever surface lenses the user hasn't encountered, or does it converge on familiar frameworks? Is there a mechanism for serendipitous lens discovery?

### 3. Feedback loop integrity: How does the system know a lens ...

- Feedback loop integrity: How does the system know a lens application was successful? What implicit and explicit signals feed back into selection quality? Is there a risk of degenerate feedback loops?

### 4. Context window as recommendation context: How does the co...

- Context window as recommendation context: How does the conversation history (short-term context) interact with the cognitive profile (long-term context) in lens selection? Are there recency bias or primacy bias risks?

### 5. Evaluation methodology: How will Auraken measure whether ...

- Evaluation methodology: How will Auraken measure whether dynamic lens selection actually outperforms random or user-selected frameworks? What offline and online evaluation approaches are planned?

### 6. Multi

- Multi-stakeholder recommendations: When Auraken expands to product recommendations and ambient suggestions, how does the system handle the tension between user value and commercial incentives?

## Severity Calibration

- **P1**: Dynamic lens selection has no cold-start strategy, causing the first 3-5 conversations to feel generic or random, destroying first-session engagement.
  - When: No fallback ranking, no onboarding signal collection, no prior-based selection for new users.
- **P2**: Lens selection converges on a narrow set of familiar frameworks, creating a filter bubble that contradicts the expanding lens repertoire goal.
  - When: No exploration mechanism, no novelty bonus, no diversity constraint in selection.
- **P2**: No measurable signal for whether a lens application was useful, making it impossible to improve selection quality over time.
  - When: Feedback is purely implicit with no calibration, or feedback collection violates the invisible feedback design principle.

When in doubt: describe the failure scenario. If it wakes someone at 3 AM, it is P0/P1. If it degrades quality over weeks, it is P2.
