## Severity Calibration

- **P0**: A worker goroutine panics on a nil DB row and there is no recover() deferred, crashing the entire service process mid-migration.
  - When: Triggered by any unexpected nil result from a query during migration, common with sparse data.
- **P0**: wg.Add is called inside the goroutine body rather than before the go statement, causing wg.Wait() to return before all workers finish.
  - When: Race condition that manifests intermittently under scheduler pressure.
- **P1**: Workers are spawned without a context, so when a migration is cancelled, workers continue running and hold DB connections open.
  - When: Occurs on any migration timeout or graceful shutdown, causing connection pool exhaustion.
