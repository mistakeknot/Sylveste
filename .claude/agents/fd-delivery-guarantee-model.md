## Task Context

Intercom is a dual-process system (Node host + Rust IronClaw daemon) where messages flow from Telegram/WhatsApp through Node, are stored in Postgres by Rust, then polled every 1 second and dispatched to Docker containers via a per-group queue. The redesign aims to replace polling and fire-and-forget HTTP callbacks with reliable event-driven IPC.

## Decision Lens

Prioritize changes that close the known fire-and-forget gaps (HttpDelegate::send_message fires a tokio task with no ack, dual-write between Node SQLite and Rust Postgres has no rollback) before optimizing for latency. A slower reliable path beats a fast lossy one.

### 3. Recovery completeness

### 4. Outbox pattern evaluation
- Evaluate the outbox pattern as a fix: would writing IPC messages to a Postgres outbox table (status=pending) before calling delegate.send_message(), then marking delivered on ack, close the HttpDelegate loss window?

### 5. LISTEN/NOTIFY delivery risk
